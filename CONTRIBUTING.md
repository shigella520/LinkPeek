# 贡献指南

## 开发流程

1. 安装 Java 17。
2. 在发起 Pull Request 之前运行 `./mvnw -B verify`。
3. 平台相关行为应尽量收敛在各自的 provider 模块内，不要泄漏到 `linkpeek-server`。

## Pull Request 要求

- 行为变更必须附带对应测试。
- 对外行为变化时，需要同步更新文档。
- 除非明确说明是破坏性变更，否则应尽量保持 API 稳定。

## 编码约定

- 统一以 Java 17 为目标版本。
- 优先选择小而明确的抽象，避免过度依赖重量级框架封装。
- “不支持的 provider” 与 “非法 URL” 必须区分清楚，不能混用错误语义。
